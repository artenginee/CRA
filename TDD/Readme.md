### git
- 1단계 : git init ~ commit
- 2단계 : checkout, git log view
- 3단계 : git branch, merge
- 4단계 : github, push, pull, clone


### command
- git log --oneline --graph --all
- git add .
- git commit -m 'test'
- git push
- git pull

### TDD
- RED : 실패하는 단계
  - todo 리스트 작성
  - 명확한 스펙 확정
- GREEN : 기능 구현
  - 어떻게든 동작하도록.
  - 딱 테스트를 통과할 만큼만 구현
  - 죄악을 저질러도 좋다 (나중에 만회를 할 것이기 때문에)
- BLUE : 리팩토링 단계


> Baby step은 꼭 거쳐야 하나?
> - 꼭 그렇지는 않지만, 작은 단계 작업을 배우면 저절로 적절한 크기의 단계 작업이 가능하다.
